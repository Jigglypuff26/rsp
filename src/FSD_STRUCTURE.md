# Feature-Sliced Design (FSD) Structure

Проект организован по методологии Feature-Sliced Design.

## Структура папок

```
src/
├── app/           # Инициализация приложения, провайдеры, роутинг
├── pages/         # Страницы приложения
├── widgets/       # Крупные самостоятельные блоки
├── features/      # Функциональность пользователя
├── entities/      # Бизнес-сущности
└── shared/        # Переиспользуемые компоненты, утилиты, UI-кит
    ├── ui/        # UI компоненты
    ├── lib/       # Утилиты и библиотеки
    ├── config/     # Конфигурация
    └── assets/    # Статические ресурсы
```

## Правила импортов

1. Импорты идут только вверх по слоям (нельзя импортировать из верхних слоев в нижние)
2. Слои: `app` → `pages` → `widgets` → `features` → `entities` → `shared`
3. Используйте относительные пути для импортов

## Примеры

```typescript
// ✅ Правильно
import { Button } from '../../shared/ui';
import { MainPage } from '../pages/main';
```
